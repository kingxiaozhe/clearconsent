# Chrome Web Store 提审文案（Dashboard 直接抄）

## 名称（≤45 字符）

ClearConsent — 透明的 Cookie 弹窗处理

## 简短描述 / summary（≤132 字符）

自动处理网站 cookie 同意弹窗，替你选「仅必要」。每次处理都有可审计的收据，零数据收集，开源。

## 详细描述（Description）

ClearConsent 自动处理你在网站上遇到的 cookie 同意弹窗——装上就忘，页面不用你手动点。

■ 三种应答策略
· 仅必要 Cookie（默认）——只同意站点运行必需的，拒绝追踪类
· 全部拒绝优先——能拒绝的全部拒绝
· 只隐藏不应答——仅移除弹窗，不替你做任何选择

■ 透明的处理收据（我们与众不同之处）
每一次处理都清清楚楚记录：隐藏了哪个元素、点了哪个按钮、用的哪种策略——而且明确标注「0 条数据离开本机」。透明日志页可随时查看、导出，永不上传。

■ 完全掌控
不想被处理的站点，一键加入信任清单，扩展彻底不介入。可随时全局暂停。

■ 隐私优先
· 零数据收集——你的浏览与处理记录只存本机
· 无支付集成、无第三方追踪、无遥测
· 开源（MIT）

内置 10 大主流 CMP 规则（OneTrust、Quantcast、Cookiebot、Sourcepoint、Usercentrics、Didomi 等），覆盖大量欧盟与国际网站。

## Privacy practices（Dashboard「隐私实践」页填写）

### 单一用途说明

自动处理网站的 cookie 同意弹窗（隐藏或按用户设定的策略应答），并向用户透明展示每次处理的动作。

### 权限用途理由（逐项，提审必填）

- **storage**：保存用户的处理偏好、信任站点清单、以及本地处理日志。全部仅存于用户本机。
- **scripting** + **host access（`<all_urls>`）**：cookie 弹窗可能出现在任意网站，扩展需在用户访问的页面上注入弹窗检测与处理逻辑。这与扩展的单一用途直接相关——没有全站访问就无法处理各站的弹窗。
- **alarms**：每 24 小时检查一次规则库更新（纯 JSON 数据），不触及任何用户数据。

### 数据收集声明

本扩展**不收集、不传输任何用户数据**。所有偏好与处理记录仅存储在用户本地设备（chrome.storage.local）。规则库更新仅从固定来源拉取 JSON 数据，请求不携带任何用户信息（无 cookie、无标识参数）。

- 是否收集个人身份信息：否
- 是否收集浏览活动：否（处理日志仅存本机，不上传）
- 是否出售数据给第三方：否
- 数据是否用于与单一用途无关的目的：否
